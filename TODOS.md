ПРОБЛЕМЫ:

при создании ордера нужно проверять есть ли о мейкера необходимый frozen

при создании ордера у мейкера замораживаются средства в заданном объёме

при выполнении ордера нужно проверять есть ли у тейкера необходимый баланс

если тейкер выкупает объёма больше чем есть в стакане, то на оставшийся объём создать ордер по последней цене

среднее арифметическое цены для тейкера отрисуем ему персонально на фронтенде

цена, объём и стоимость должны быть больше нуля

если в книге заявок открыта позиция, которая ничем не обеспечана (у маркет мейкера отсутсвует замороженная сумма, которая бы покрывала объём этого ордера и в целом объём всех открытых этим мейкером ордеров ), то такой ситуации вообще не должно быть.

во время совершения сделки(take) слишком поздно проверять наличие замороженных средств у мейкера, которые обеспечивают эту позицию

функция swap работает только с frozen средствами мейкера

все открытые ордера зашиты в frozen

маркет мейкер продаёт только то что frozen

тейкер не может выкупить больше чем объём ордера

порядок действий:

1. трейдеры пополняют свой баланс (симулируем, тупо записываем им баланс)
2. маркет мейкер открывает ордер (make, )
3. тейкер забирает по объёму (или по стоимости )

в конечном итоге тейкер забирает у мейкера объём, который заморожен
